v1.0 · 2026.01

把"陪伴"
拆成骨架 Disassembling companionship

v1 把"陪伴"分成两件可以独立工程化的事: 一是记忆 — 让她记得你说过什么; 二是关系 — 让她跟你的距离会变化。 v1 split "companionship" into two independently engineerable pieces: memory — she remembers what you've said; and relationship — your distance to her actually changes.

技术上, FastAPI + React + Ollama (qwen2.5:7b) + SQLite + ChromaDB; 故事上, 这是地基。 Stack: FastAPI + React + Ollama (qwen2.5:7b) + SQLite + ChromaDB. In narrative terms — this is the foundation.

三层记忆 three-layer memory

L1

结构化档案Structured profile SQLite

职业、城市、喜好 — 每条带 importance (1-10) 用户可改。 Job, city, preferences — each carries a user-adjustable importance (1-10).

L2

重要事件Important events SQLite

生日、吵架、第一次见面 — 带 emotional_weight (0-1), 注入 system prompt 时按权重排序。 Birthdays, fights, first meets — weighted (0-1), ordered by emotional weight when injected.

L3

向量 RAGVector RAG ChromaDB

每条用户消息做语义搜索, 召回最相关的 4 段历史摘要; 每 20 轮异步触发摘要提取。 Each message does a semantic lookup, pulls the 4 most relevant past summaries; extraction runs async every 20 turns.

三层叠加进 system prompt — 重要的东西先说, 相关的东西及时浮上来。 Three layers stack into the system prompt — important things first, relevant things surface in time.

给她情绪 Give her feelings

v1 的她, 是个善于回答的人。v2 的她, 是个真的会反应的人。 v1 was someone good at answering. v2 is someone who actually reacts.

Model upgraded to qwen2.5:14b.

v2.2 · 节奏rhythm

边写边发, 像真人发微信 Sentence by sentence, like a real text

原版等 LLM 写完一整段才发出 — 首句出现要 4-8 秒, 不像活人。v2.2 后端改成边生成边按句号发, 前端拆成 2-4 条气泡。你说话不会被她"挡住" — 她还没说完, 你发新消息, 旧回复会被立即打断。 The original waited for a full reply — 4-8 second time-to-first-word, doesn't feel alive. v2.2 streams sentence by sentence, the frontend splits into 2-4 bubbles. You're never blocked — send a new message mid-reply, and her old response is interrupted instantly.

今天搞砸了 I messed up today
咋了 what happened
说来听听 tell me
先深呼吸 take a breath first
逐句延迟 = min(0.4 + 字数·0.03, 2.2s) per-segment delay = min(0.4 + len·0.03, 2.2s)

v3.0 · 第二次转向 · 开源second pivot · open source

每个人都能
在自己电脑上跑 Anyone can run her on their own machine

v3 之前, 她只跑在我的笔记本上。v3 把所有硬编码绝对路径清掉、JWT 密钥变成"缺省自动生成"、nginx 配置变成模板、Cloudflared 公网隧道变可选 — 任何人 clone 下来都能跑。 Before v3, she only ran on my laptop. v3 stripped every hardcoded absolute path, made the JWT key auto-generate per deploy, turned the nginx config into a template, made the Cloudflared tunnel optional — anyone can clone and run.

同时建了一个落地页 (docs/index.html), 部署到 GitHub Pages 当产品官网, 含 8 个特性卡 / 3 步安装 / 关系阶段对比 / 7 条 FAQ。v3.1.1 上了官网双语切换。 In parallel I built a landing page (docs/index.html) on GitHub Pages — 8 feature cards / 3-step install / relationship-stage comparison / 7 FAQs. v3.1.1 added the site's CN/EN switch.

v4.0 · 第三次转向 · 国际化third pivot · i18n

让她自己说英文 —
不是 UI 英文 + AI 中文的分裂 Let her speak English —
not English UI on top of a Chinese AI

前端自建轻量 i18n (无第三方库), 后端给 personality 加 language 字段, personality_engine 与 emotion_engine 的所有 prompt 出双语版 (关系阶段描述 / 亲密度规则 / 说话方式 / 身份指令 / 年龄风格 / 冷战记仇 / 依赖风格), 敏感词正则同时匹配中英输入, 英文不是机翻 — 是重写。 A lightweight in-house i18n on the frontend (no library). Backend adds a personality.language field. Every prompt in personality_engine and emotion_engine got a bilingual variant — stage descriptions, intimacy rules, speaking style, identity defense, age tone, cold-war and grudge. Filter regex matches both languages. The English copy is rewritten, not auto-translated.
